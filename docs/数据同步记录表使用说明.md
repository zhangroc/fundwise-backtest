# 数据同步记录表使用说明

## 表结构说明

### 1. data_sync_log - 数据同步日志表

记录每次数据同步的详细信息。

| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT | 自增主键 |
| sync_type | VARCHAR(50) | 同步类型：fund_list/nav_data/quality_upgrade/full_sync |
| sync_source | VARCHAR(100) | 数据来源：akshare/tushare/manual/algorithm |
| sync_status | ENUM | 同步状态：pending/running/success/failed/partial |
| fund_count | INT | 涉及基金数量 |
| fund_codes | TEXT | 基金代码列表（JSON 数组） |
| date_start | DATE | 数据开始日期 |
| date_end | DATE | 数据结束日期 |
| records_inserted | INT | 新增记录数 |
| records_updated | INT | 更新记录数 |
| records_failed | INT | 失败记录数 |
| start_time | DATETIME | 开始时间 |
| end_time | DATETIME | 结束时间 |
| duration_seconds | INT | 耗时（秒） |
| error_message | TEXT | 错误信息 |
| operator | VARCHAR(100) | 操作者 |
| script_path | VARCHAR(255) | 执行脚本路径 |
| batch_id | VARCHAR(100) | 批次 ID |
| remarks | TEXT | 备注说明 |

### 2. data_quality_snapshot - 数据质量快照表

定期记录数据质量的变化趋势。

| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT | 自增主键 |
| snapshot_date | DATE | 快照日期 |
| excellent_count | INT | 优质数据基金数（⭐⭐⭐⭐⭐） |
| good_count | INT | 良好数据基金数（⭐⭐⭐⭐） |
| fair_count | INT | 一般数据基金数（⭐⭐⭐） |
| poor_count | INT | 无数据基金数（⭐） |
| total_funds | INT | 基金总数 |
| total_nav_records | BIGINT | 净值记录总数 |
| avg_records_per_fund | DECIMAL | 平均单只记录数 |

## 使用方法

### Python 脚本示例

```python
from sync_logger import DataSyncLogger

logger = DataSyncLogger()

try:
    # 开始同步
    logger.start_sync(
        sync_type='nav_data',
        sync_source='akshare',
        fund_count=100,
        date_start='2020-01-01',
        date_end='2026-02-27',
        operator='import_nav_data.py',
        batch_id='BATCH-20260227-002',
        remarks='导入 100 只基金的历史净值数据'
    )
    
    # 同步过程中更新进度
    logger.update_sync_progress(
        records_inserted=10000,
        records_failed=2
    )
    
    # 完成同步
    logger.complete_sync(
        records_inserted=36500,
        records_failed=2,
        status='partial'
    )
    
    # 创建质量快照
    logger.create_quality_snapshot(remarks='定期快照')
    
except Exception as e:
    logger.fail_sync(str(e))
finally:
    logger.close()
```

### SQL 查询示例

#### 查询最近的同步记录
```sql
SELECT 
    sync_type,
    sync_status,
    fund_count,
    records_inserted,
    start_time,
    duration_seconds
FROM data_sync_log
ORDER BY created_at DESC
LIMIT 10;
```

#### 查询数据质量变化趋势
```sql
SELECT 
    snapshot_date,
    excellent_count as '优质',
    good_count as '良好',
    poor_count as '无数据',
    total_nav_records as '净值记录'
FROM data_quality_snapshot
ORDER BY snapshot_date DESC;
```

#### 统计同步成功率
```sql
SELECT 
    sync_type,
    COUNT(*) as total,
    SUM(CASE WHEN sync_status = 'success' THEN 1 ELSE 0 END) as success,
    ROUND(SUM(CASE WHEN sync_status = 'success' THEN 1 ELSE 0 END) * 100.0 / COUNT(*), 2) as success_rate
FROM data_sync_log
GROUP BY sync_type;
```

## 最佳实践

1. **每次数据同步都记录日志**
2. **定期创建质量快照**（建议每日/每周）
3. **使用 batch_id 关联相关同步**
4. **详细记录错误信息**便于排查
5. **监控同步成功率**及时发现问题
